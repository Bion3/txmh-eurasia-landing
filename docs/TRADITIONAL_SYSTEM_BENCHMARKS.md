# 传统 CRM / 客户管理 / 供应商管理借鉴点

本文档用于把传统业务系统的成熟做法转成当前物流系统的后续优化方向。

## 1. CRM 可借鉴能力

- 客户生命周期：线索、潜客、报价中、成交、沉睡、流失。
- 客户分层：按成交、报价、货量、毛利和复购周期划分 A/B/C。
- 联系人网络：一个客户多个联系人，记录职务、偏好渠道和决策角色。
- 跟进节奏：首响 SLA、二次触达、复购提醒、沉睡唤醒。
- 销售漏斗：从访问、询盘、报价、订单、回款追踪转化率。

当前已覆盖：线索池、客户列表、客户分层、复购/唤醒清单、客户去重/合并候选、客户商业治理、报价跟进、SLA 队列，以及总览页的传统系统借鉴雷达。

下一步建议：客户时间线继续增强、真实合并审批/审计、联系人角色、客户等级规则持久化和复购预测。

## 2. 传统客户管理系统可借鉴能力

- 主数据完整度：税号、国家、行业、地址、账期、联系人必须可见。
- 资料缺口提示：缺资料会影响报价、订单、财务时，应提前提示。
- 客户画像：常走线路、偏好运输方式、货物类型、价格敏感度。
- 任务闭环：每次沟通都要有下一步动作、负责人和日期。

当前已覆盖：资料缺口、联系人维护、客户快照、复购/唤醒动作、重复客户候选、合并检查清单、合同账期/合并审批/等级规则治理。

下一步建议：客户详情深链后继续补附件上传、合同/账期字段持久化和真实合并审批记录。

## 3. 供应商管理系统可借鉴能力

- 供应商档案：类型、国家、联系人、邮箱、电话、账期、币种、状态。
- 服务覆盖：供应商可服务的运输方式、线路、港口和货型。
- 费率有效期：过期、临期、草稿和未绑定供应商要变成待办。
- 供应商分层：核心供应商、备选供应商、高风险供应商。
- 对账风险：账期、币种、发票、付款状态和异常应与财务打通。

当前已覆盖：供应商列表、费率表、费率项、费率健康/补价指挥台、供应商 KPI / 风险画像、供应商附件/响应/对账付款治理、费率版本/生效审批准备度。

本轮新增：成本中心内的供应商运营台，会按供应商聚合费率状态、服务覆盖、档案缺口和下一步动作，并支持复制供应商行动清单。

本轮继续新增：供应商 KPI / 风险画像，会按费率新鲜度、档案完整度、服务覆盖、财务准备和异常压力生成 KPI 分数、等级、风险原因和下一步采购动作。

本轮继续新增：供应商附件 / 响应 / 对账付款，会把合同、执照、银行账户、报价附件、服务联系人、报价响应时效、应付逾期和对账争议合成详情治理分，并可复制供应商详情治理清单。

下一步建议：供应商真实附件上传、报价响应自动采集、对账付款回写和真实费率审批流。

## 4. 本轮新增：传统系统借鉴雷达

系统总览新增“传统系统借鉴雷达”，把传统 CRM、客户管理系统、供应商管理 / SRM、报价与合同系统、TMS / 履约系统、ERP / 财务系统的成熟能力映射到当前项目：

- 每一行展示来源系统、成熟能力、已借鉴到项目的能力、当前业务信号和下一步优化。
- 当前信号会结合线索待跟进、高风险客户、低毛利报价、订单待成本和待收款等工作台指标，标记 P1/P2/P3。
- 点击来源系统可直接进入对应模块，方便从对标分析切回实际处理。
- 交付脚本 `npm run check:delivery` 已校验该雷达存在，避免后续改动误删这层经营视角。

## 5. 当前项目优化排序

1. 客户/报价/订单详情页组件拆分，增强深链直达后的权限提示、加载骨架和空状态。
2. 供应商详情页、供应商附件、报价响应时效、对账/付款风险和真实费率审批流。
3. 报价审批角色配置、版本详情页、正式 PDF Storage 归档和邮件发送回执。
4. 真实附件上传、异常责任流持久化、客户轨迹分享链接和签收回单 Storage 归档。
5. ERP/用友/金蝶真实 API 对接、导出回执前端队列和失败重试处理页。
