# 项目系统检查报告

## 结论

项目已经包含客户管理、订单管理、营销获客、报价中心、成本中心和财务结算的基础架构，但之前的使用逻辑确实不顺。主要问题不是“完全没有这些系统”，而是：

- 所有模块被放在一个 `System Workspace` 里，像流程演示，不像独立业务系统。
- 默认进入线索池，没有系统总览，用户不知道应该从哪里开始。
- 客户管理之前只有客户详情，没有客户列表入口。
- 订单管理之前偏订单详情，没有清楚的“列表/录单/详情”结构。
- 营销获客之前有线索池和数据表，但邮件跟进、广告来源、活动管理和官网快速询盘入口不够明确。
- 成本中心已参与报价计算，并已补基础费率维护页面。

本轮已先修正入口逻辑：新增系统总览、重排模块导航、补客户列表入口、补订单列表入口、补官网快速询盘、获客来源和二次邮件跟进入口、保留订单手工录单能力。

当前业务范围已进一步聚焦到中欧、中亚、中俄及欧亚大陆陆路通道，优先覆盖班列拼箱 LCL、班列整柜 FCL 和卡车业务；同时已整理 `admin`、`manager`、`sales`、`marketing`、`ops`、`finance`、`viewer`、`anon` 的登录权限分类，详见 `docs/EURASIA_BUSINESS_SCOPE_AND_PERMISSIONS.md`。

## 现在推荐的使用路径

### 路径 A：标准销售链路

1. 进入 `系统总览`
2. 进入 `营销获客`
3. 选择线索
4. 转客户或直接创建报价
5. 在 `报价中心` 核价并保存报价
6. 标记发送或提交审批
7. 报价转订单
8. 在 `订单管理` 补订舱、路线和货物信息
9. 生成应收/应付进入 `财务结算`

### 路径 B：已有客户报价

1. 进入 `客户管理`
2. 找到客户或直接新增客户
3. 补齐联系人、国家、行业、税号等关键资料
4. 按客户分层安排二次邮件/复购跟进
5. 点击创建报价
6. 报价保存后转订单

### 路径 C：直接录单

1. 进入 `订单管理`
2. 在手工录单区域填写客户、订舱号、路线、体积、重量、收入、成本
3. 保存订单
4. 生成应收/应付进入财务

## 为什么之前使用逻辑奇怪

1. 缺少系统总览：用户一进来就是线索池，不知道这是 CRM、TMS 还是后台。
2. 模块命名偏开发：`Lead Pool`, `Customer Workspace`, `Order Detail` 不像中文业务系统。
3. 客户页依赖上一步：没有从线索转客户时，客户管理页面缺少自然入口。
4. 订单页职责混乱：既叫订单详情，又承担录单、财务生成入口。
5. 营销获客没有独立表达：表结构有 campaign/email task，但 UI 只体现线索池。
6. 成本中心之前隐藏在报价逻辑里；现在已补独立入口，后续继续补导入、版本和审批。
